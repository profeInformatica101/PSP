RA1 · Process / ProcessBuilder (Carrusel 0–11)

Objetivo: desarrollar aplicaciones con varios procesos aplicando principios de concurrencia y paralelismo. Cada ejercicio explica métodos Java, comandos del SO y el criterio RA1 al que contribuye.

0) Utilidades y portabilidad de comandos

Base común para todos los ejercicios (portabilidad Linux/Windows).

// Utils.java
  import java.util.*;
  public class Utils {
    public static boolean isWindows() {
      return System.getProperty("os.name").toLowerCase().contains("win");
    }
    public static List<String> sh(String cmd) {
      return isWindows() ? Arrays.asList("cmd","/c",cmd)
                        : Arrays.asList("sh","-c",cmd);
    }
  }

1) Ciclo básico: start · waitFor · exitValue

Métodos Java:

  • new ProcessBuilder(List<String>): define comando.
  • start(): lanza el subproceso.
  • waitFor(): bloquea hasta finalizar y devuelve RC.
  • exitValue(): código de salida (0 OK, ≠0 error).

Comandos SO: echo hola (Linux) · cmd /c echo hola (Win)

// Ej1_Basico.java
  import java.util.*;
  public class Ej1_Basico {
    public static void main(String[] args) throws Exception {
      Process p = new ProcessBuilder(Utils.sh("echo hola")).start();
      int rc = p.waitFor();
      System.out.println("RC=" + rc);
    }
  }

2) E/S de procesos: STDOUT y STDERR

Métodos Java:

  • getInputStream(): lee STDOUT.
  • getErrorStream(): lee STDERR.
  • BufferedReader: lectura línea a línea (requiere cierre manual con close()).

Comandos SO: java -version (suele ir a STDERR), echo hola (STDOUT)

// Ej2_LeerSalida.java
  import java.io.*; 
  import java.util.*;

  public class Ej2_LeerSalida {
    public static void main(String[] args) {
      // Para convertir bytes en texto:
      // - InputStreamReader → convierte bytes a caracteres.
      // - BufferedReader → permite leer líneas completas y cerrar el recurso.

      try {
        BufferedReader br;

        // ==== PRIMER PROCESO: echo hola ====
        Process p1 = new ProcessBuilder(Utils.sh("echo hola")).start();
        br = new BufferedReader(new InputStreamReader(p1.getInputStream()));
        System.out.println("STDOUT: " + br.readLine());
        br.close();
        p1.waitFor();

        // ==== SEGUNDO PROCESO: java -version (va a STDERR) ====
        Process p2 = new ProcessBuilder(Utils.sh("java -version")).start();
        br = new BufferedReader(new InputStreamReader(p2.getErrorStream()));
        String line;
        while ((line = br.readLine()) != null) {
          System.out.println(line);
        }
        br.close();
        p2.waitFor();

      } catch (IOException e) {
        // Error de entrada/salida: no se pudo leer la salida del proceso
        System.err.println("Error de E/S: " + e.getMessage());
      } catch (InterruptedException e) {
        // Otro hilo interrumpió la espera del proceso
        System.err.println("El proceso fue interrumpido");
      }
    }
  }

3) Redirecciones: unir flujos y enviar a fichero

Métodos Java:

  • redirectErrorStream(true): STDERR → STDOUT.
  • redirectOutput(File), redirectError(File): a ficheros.
  • inheritIO(): hereda consola actual (útil para depurar).
// Ej3_Redir.java
  import java.io.*; import java.util.*;
  public class Ej3_Redir {
    public static void main(String[] args) throws Exception {
      File out = new File("salida.txt"), err = new File("errores.txt");

      ProcessBuilder a = new ProcessBuilder(Utils.sh("java -version"));
      a.redirectErrorStream(true).redirectOutput(out).start().waitFor();

      ProcessBuilder b = new ProcessBuilder(Utils.sh(Utils.isWindows()? "dir" : "ls -la"));
      b.redirectOutput(out).redirectError(err).start().waitFor();

      new ProcessBuilder(Utils.sh("echo depurando con inheritIO"))
        .inheritIO().start().waitFor();
    }
  }

Métodos Java:

      p.waitFor();
    }
  }

5) Envío por STDIN (IPC simple) → sort con entrada de teclado

Idea clave: nuestro programa Java escribe líneas al STDIN del proceso sort, y después lee el resultado ya ordenado desde su STDOUT. Aquí añadimos también una fruta introducida por el usuario.

Métodos Java:

  • getOutputStream(): canal para escribir al proceso hijo (su STDIN) → usamos BufferedWriter.
  • getInputStream(): canal para leer del proceso hijo (su STDOUT) → usamos BufferedReader.
  • waitFor(): sincroniza hasta que el proceso termine.

Comando SO: sort (disponible en Linux y Windows).

// Ej5_Stdin.java (con entrada por teclado)
  package psp;

  import java.io.BufferedReader;
  import java.io.BufferedWriter;
  import java.io.InputStreamReader;
  import java.io.OutputStreamWriter;
  import java.util.Scanner;

  public class Ej5_Stdin {

      public static void main(String[] args) throws Exception {
          // 1) Preparamos lectura por teclado (consola)
          Scanner sc = new Scanner(System.in);

          // 2) Lanzamos el proceso "sort" (ordenará las líneas de texto que reciba por STDIN)
          Process p = new ProcessBuilder(Utils.sh("sort")).start();

          try {
              // 3) Pedimos un dato al usuario para integrarlo en la lista
              System.out.print("Escribe una fruta: ");
              String fruta = sc.nextLine();

              // 4) ENVIAR datos al proceso (STDIN):
              //    - Escribimos varias frutas + la fruta introducida por el usuario.
              //    - Cerramos el Writer para indicar "fin de datos" al proceso sort.
              BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(p.getOutputStream()));
              bw.write("pera\nmanzana\nuva\n" + fruta + "\n");
              bw.close(); // <-- importante para que sort empiece a procesar

              // 5) LEER el resultado ordenado del proceso (STDOUT):
              //    - Consumimos línea a línea hasta que no haya más datos.
              BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()));
              String line;
              System.out.println("\nListado ordenado:");
              while ((line = br.readLine()) != null) {
                  System.out.println(line);
              }
              br.close();

              // 6) Sincronizamos: esperamos a que el proceso termine
              p.waitFor();

          } catch (Exception e) {
              // Captura genérica para simplificar en esta práctica
              System.err.println("Ocurrió un error: " + e.getMessage());
          } finally {
              // 7) Buenas prácticas: cerrar recursos creados por nosotros
              sc.close();
          }
      }
  }

Notas rápidas para el alumnado:

  • Si no cierras el BufferedWriter, sort se queda esperando más datos y no devuelve nada.
  • STDIN del proceso hijo ⇢ lo obtenemos con p.getOutputStream() (desde Java “es salida”, hacia el hijo es “entrada”).
  • STDOUT del proceso hijo ⇢ lo leemos con p.getInputStream().

6) Encadenado (pipe manual): productor → consumidor

Idea clave: simulamos p1 | p2 desde Java. El proceso p1 produce texto por su STDOUT y lo canalizamos al STDIN de p2 (que ordena con sort). Es imprescindible cerrar el STDIN de p2 para que procese (EOF).

Métodos Java:

  • getInputStream() y getOutputStream() para acceder a STDOUT/STDIN.
  • transferTo(...) para canalizar bytes entre streams.
  • waitFor() para sincronizar finalización.

Comandos SO: echo/printf → sort (Linux/Windows).

// Ej6_PipeManual.java (pipeline p1 | p2 con cierre explícito de STDIN)
  import java.io.*;
  import java.util.*;

  public class Ej6_PipeManual {
    public static void main(String[] args) throws Exception {

      // 1) Comando productor (portabilidad):
      //    - Windows: varios 'echo' encadenados (&)
      //    - Linux: printf con saltos de línea
      String echoCmd = Utils.isWindows()
          ? "(echo a&echo c&echo b)"
          : "printf \"a\nc\nb\n\"";

      // 2) Lanzamos procesos:
      //    p1 = productor (genera líneas)   p2 = consumidor (sort)
      Process p1 = new ProcessBuilder(Utils.sh(echoCmd)).start();
      Process p2 = new ProcessBuilder(Utils.sh("sort")).start();

      // Recursos para cierre manual en finally (pedagógico)
      InputStream  outDeP1   = null;   // STDOUT de p1
      OutputStream inDeP2    = null;   // STDIN  de p2
      BufferedReader brDeP2  = null;   // STDOUT de p2 (lectura final)

      try {
        // 3) Conectar productor → consumidor
        outDeP1 = p1.getInputStream();     // lo que p1 escribe
        inDeP2  = p2.getOutputStream();    // lo que p2 lee

        // Canalizamos todos los bytes del productor al consumidor
        outDeP1.transferTo(inDeP2);

        // *** CLAVE: cerrar el STDIN de p2 ***
        // Sin este close(), 'sort' espera más datos y no devuelve resultado.
        inDeP2.close();

        // 4) (Opcional) sincronizar con el productor
        p1.waitFor();

        // 5) Leer salida ordenada del consumidor (STDOUT de p2)
        brDeP2 = new BufferedReader(new InputStreamReader(p2.getInputStream()));
        String line;
        System.out.println("Resultado ordenado (p1 | p2):");
        while ((line = brDeP2.readLine()) != null) {
          System.out.println(line);
        }

        // 6) Sincronizar con el consumidor
        p2.waitFor();

      } catch (IOException e) {
        System.err.println("Error de E/S en el pipe: " + e.getMessage());
      } catch (InterruptedException e) {
        System.err.println("Interrupción mientras esperábamos a los procesos.");
      } finally {
        // 7) Buenas prácticas: cerrar todo lo abierto por nosotros
        try { if (outDeP1  != null) outDeP1.close(); }  catch (IOException ignored) {}
        try { if (inDeP2   != null) inDeP2.close(); }   catch (IOException ignored) {}
        try { if (brDeP2   != null) brDeP2.close(); }   catch (IOException ignored) {}
      }
    }
  }

Notas rápidas para el alumnado:

  • Orden recomendado: canalizar → cerrar STDIN de p2 → leer STDOUT de p2 → waitFor().
  • transferTo no cierra flujos por ti; el cierre del canal de entrada de p2 es tu responsabilidad.
  • Esto replica el pipe del shell (cmd | sort) pero controlado desde Java (IPC).

7) Lifecycle: waitFor(timeout), destroy() y destroyForcibly()

Idea clave: controlamos el ciclo de vida de un subproceso. Primero esperamos con waitFor(t, unit); si no termina a tiempo, probamos una terminación suave con destroy(). Si persiste, aplicamos una terminación forzosa con destroyForcibly().

Métodos Java:

  • waitFor(long, TimeUnit): espera con timeout (no bloquea indefinidamente).
  • destroy(): intenta terminar de forma ordenada (en Unix envía SIGTERM).
  • destroyForcibly(): fuerza la terminación si sigue vivo.
  • isAlive(), exitValue(): estado y código de salida del proceso.

Comandos SO de ejemplo: sleep/timeout para simular tareas lentas.

// Ej7_Timeout.java
import java.util.concurrent.*;

public class Ej7_Timeout {

  // Tiempo máximo que esperamos antes de considerar que hay "timeout"
  private static final long TIMEOUT_SEG = 1; 

  // Margen de gracia que damos tras un destroy() antes de forzar con destroyForcibly()
  private static final long GRACE_SEG   = 1; 

  public static void main(String[] args) throws Exception {
    if (Utils.isWindows()) {
      // Aviso didáctico: en Windows no hay señales tipo SIGTERM.
      // Allí destroy() suele comportarse ya como una terminación forzosa.
      System.out.println("⚠ Nota: En Windows, destroy() suele comportarse como forzoso. Usa WSL/VM para ver los 3 caminos.");
    }

    // === ESCENARIO 1: el proceso termina solo antes del timeout
    ejecutarEscenario("1) Termina a tiempo (no hay timeout)",
        comandoTerminaATiempo(), TIMEOUT_SEG);

    // === ESCENARIO 2: el proceso no termina a tiempo pero sí obedece al destroy()
    ejecutarEscenario("2) Timeout + destroy() (obedece SIGTERM)",
        comandoObedeceTerminacionSuave(), TIMEOUT_SEG);

    // === ESCENARIO 3: el proceso no termina a tiempo y tampoco obedece a destroy()
    // se requiere destroyForcibly()
    ejecutarEscenario("3) Timeout + destroyForcibly() (ignora SIGTERM)",
        comandoIgnoraTerminacionSuave(), TIMEOUT_SEG);
  }

  /**
   * Ejecuta un comando como subproceso y gestiona:
   *  - si acaba antes del timeout → OK
   *  - si no, se intenta finalizar con destroy() (suave)
   *  - si tampoco muere → se usa destroyForcibly() (forzado)
   */
  private static void ejecutarEscenario(String titulo, String comando, long timeoutSeg) throws Exception {
    System.out.println("\n=== " + titulo + " ===");
    System.out.println("CMD: " + comando + " | timeout=" + timeoutSeg + "s");

    // Lanzamos el proceso
    Process p = new ProcessBuilder(Utils.sh(comando)).start();

    // Esperamos al proceso con un tiempo máximo (timeoutSeg segundos)
    boolean terminoATiempo = p.waitFor(timeoutSeg, TimeUnit.SECONDS);

    if (!terminoATiempo) {
      // Caso 1: no ha terminado → aplicamos destroy()
      System.out.println("Timeout → destroy() (terminación suave)");
      p.destroy();

      // Le damos un margen para que responda al destroy()
      boolean murioTrasDestroy = p.waitFor(GRACE_SEG, TimeUnit.SECONDS);

      // Si aún sigue vivo, aplicamos la terminación forzosa
      if (!murioTrasDestroy && p.isAlive()) {
        System.out.println("Aún vivo → destroyForcibly() (terminación forzosa)");
        p.destroyForcibly();
      }
    }

    // Estado final del proceso
    System.out.println("Fin. isAlive=" + p.isAlive());
    if (!p.isAlive()) {
      // exitValue = 0 suele indicar final correcto
      System.out.println("exitValue=" + p.exitValue());
    }
  }

  // ==== Comandos por escenario ====

  /**
   * Escenario 1: Proceso corto que acaba antes del timeout.
   */
  private static String comandoTerminaATiempo() {
    return Utils.isWindows()
        ? "timeout /t 1 >nul"  // Windows
        : "sleep 1";             // Linux/macOS
  }

  /**
   * Escenario 2: Proceso largo que obedece a la terminación suave.
   * - En Linux: capturamos SIGTERM y salimos enseguida.
   * - En Windows: usamos un sleep largo (pero destroy() lo termina).
   */
  private static String comandoObedeceTerminacionSuave() {
    return Utils.isWindows()
        ? "powershell -Command \"Start-Sleep -Seconds 30\""
        : "sh -c 'trap \"echo TERM recibido; exit 0\" TERM; sleep 30'";
  }

  /**
   * Escenario 3: Proceso largo que ignora la señal suave (SIGTERM).
   * Esto obliga a usar destroyForcibly().
   */
  private static String comandoIgnoraTerminacionSuave() {
    return Utils.isWindows()
        ? "powershell -Command \"Start-Sleep -Seconds 30\""
        : "sh -c 'trap \"\" TERM; sleep 30'";
  }
}

Notas rápidas para el alumnado:

  • Timeout: waitFor(t, unit) evita bloqueos indefinidos.
  • Suave vs. forzoso: destroy() intenta salida ordenada; si el proceso ignora la señal, usamos destroyForcibly().
  • Windows vs. Unix: en Windows no hay SIGTERM; la demo completa de los tres caminos se ve mejor en WSL o Linux/macOS.
  • Diagnóstico: imprime isAlive y exitValue para comprobar cómo terminó el subproceso.

8) Inspección y callback: ProcessHandle + onExit()

Métodos Java:

  • p.pid(), p.info() (command, args, startInstant).
  • onExit() → CompletableFuture<Process> (no bloqueante).
// Ej8_InfoOnExit.java
  import java.util.*; import java.time.*;
  public class Ej8_InfoOnExit {
    public static void main(String[] args) throws Exception {
      Process p = new ProcessBuilder(Utils.sh(Utils.isWindows()? "timeout /t 2 >nul" : "sleep 2")).start();
      System.out.println("PID=" + p.pid());
      p.info().command().ifPresent(c -> System.out.println("cmd=" + c));
      p.onExit().thenAccept(proc -> System.out.println("RC=" + proc.exitValue()));
      p.waitFor();
    }
  }

9) Paralelismo: ejecutar múltiples procesos y sincronizar resultados

Métodos Java: ExecutorService, Callable, Future.get(), waitFor()

Comando SO (ejemplo): echo tarea-i (sustituye por tu CLI real)

// Ej9_Paralelo.java
  import java.util.*; import java.util.concurrent.*; import java.io.*;
  public class Ej9_Paralelo {
    public static void main(String[] args) throws Exception {
      ExecutorService pool = Executors.newFixedThreadPool(4);
      List<Callable<Integer>> tareas = new ArrayList<>();
      for (int i=1; i<=6; i++) {
        final int id = i;
        tareas.add(() -> {
          Process p = new ProcessBuilder(Utils.sh("echo tarea-" + id)).start();
          try (BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()))) {
            System.out.println("OUT[" + id + "]: " + br.readLine());
          }
          return p.waitFor(); // sincroniza y devuelve RC
        });
      }
      List<Future<Integer>> r = pool.invokeAll(tareas);
      pool.shutdown();
      System.out.println("RCs: " + r.stream().map(f -> {
        try { return String.valueOf(f.get()); } catch(Exception e){ return "ERR"; }
      }).toList());
    }
  }

10) Monitorización de procesos desde Java

Métodos Java: lectura de salida, composición de comandos portables.

Comandos SO:

  • Linux: ps aux, pgrep -f PATRON, top -b -n 1 | head -n 5, kill PID.
  • Windows: tasklist, findstr /i PATRON, taskkill /PID PID /F.
// Ej10_Monitor.java
  import java.io.*; import java.util.*;
  public class Ej10_Monitor {
    static void run(String cmd) throws Exception {
      Process p = new ProcessBuilder(Utils.sh(cmd)).start();
      try (BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()))) {
        String line; while ((line=br.readLine())!=null) System.out.println(line);
      }
      p.waitFor();
    }
    public static void main(String[] args) throws Exception {
      if (Utils.isWindows()) {
        run("tasklist | findstr /i java");
      } else {
        run("ps aux | grep -i java | grep -v grep");
        run("top -b -n 1 | head -n 5");
      }
    }
  }

11) Servicios del sistema (arranque/estado)

Métodos Java: ProcessBuilder + lectura de salida; manejo de códigos de retorno.

Comandos SO:

  • Linux (systemd): systemctl status|start|stop nombre; fallback: service nombre status.
  • Windows: sc query "Nombre", sc start|stop "Nombre".
// Ej11_Servicios.java
  import java.io.*; import java.util.*;
  public class Ej11_Servicios {
    static void print(Process p) throws Exception {
      try (BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()))) {
        String s; while((s=br.readLine())!=null) System.out.println(s);
      }
      p.waitFor();
    }
    public static void main(String[] args) throws Exception {
      String nombre = (args.length>0? args[0] : (Utils.isWindows()? "Dnscache" : "cron"));
      if (Utils.isWindows()) {
        print(new ProcessBuilder(Utils.sh("sc query \"" + nombre + "\"")).start());
      } else {
        Process p = new ProcessBuilder(Utils.sh("systemctl status " + nombre)).start();
        int rc = p.waitFor();
        if (rc!=0) { System.out.println("(fallback) service"); print(new ProcessBuilder(Utils.sh("service " + nombre + " status")).start()); }
        else { print(new ProcessBuilder(Utils.sh("systemctl status " + nombre)).start()); }
      }
    }
  }
